My Voyage

During the great plague, I first set sail into the seas of Web Development, revisiting the arts of HTML, CSS, and JavaScript. With plenty of time on my hands, I spent several months strengthening my understanding of the fundamentals and charting the beginnings of my journey.

As life gradually returned to normal and duties ashore grew heavier, my studies slowed. Long days of work often left little energy for further exploration, and my voyage drifted from its original course

Yet about a year ago, the call of a new horizon grew too strong to ignore. Seeking fresh challenges and a renewed sense of accomplishment, I committed myself to learning programming once more. Since then, I have focused on steady progress, believing that even the smallest step forward each day brings me closer to my destination.

This website serves as my captain's log—a record of the knowledge I've gathered, the projects I've built, and the milestones I've reached along the way. It is both a showcase of my journey and a reminder that every great voyage begins with a single course set toward the horizon

October - November 2025
The Beginning

Theme: Single-Page E-commerce Website

Technologies Used: HTML CSS JavaScript

Overview: This was my first single-page E-commerce website built using HTML, CSS, and vanilla JavaScript. It was also my first real web development project. Since I had just returned to studying web development, I found it challenging to relearn and implement Flexbox and CSS Grid effectively. For product ordering, I used localStorage to store cart data. During the development process, I relied on AI tools to help me understand the basic project structure and concepts. Although I followed tutorials on YouTube and other websites, I sometimes struggled to fully understand the explanations, so AI served as an additional learning resource. Looking back, I can clearly see areas where I was still a beginner, but this project helped me build a strong foundation.

Techniques Learned: Flexbox, CSS Grid, localStorage, floating navigation, signup/login functionality, adding products to a cart, removing products from a cart, and calculating the total order price.

Challenging Aspects: Implementing responsive design, managing state with localStorage, passing data between multiple functions, and implementing the signup/login functionality.

Difficulty: Hard (I stopped following tutorials and challenged myself to build the project independently.)

Technologies Used: React AWS

I plan to revisit AWS in the future when I am ready to deploy larger and more advanced projects.

The Confusions

Theme: Multi-Page Coffee Shop Website to React

Technologies Used: HTML CSS JavaScript React

Overview: I thought I had a good grasp of how React worked after converting the single-page E-commerce website, so I decided to take on the challenge of converting the more complex multi-page coffee shop website into a React application. But, oh boy, I was wrong. This project was significantly more difficult than the single-page E-commerce project because of its increased complexity and the need to manage state and props across multiple components and pages. I found myself struggling to understand how to structure the application effectively in React and how to manage the flow of data between components. As usual, I spent more time watching tutorials and researching how to implement certain features in React than actually working on the project itself. Although it took a lot of trial and error (and some AI help), I eventually managed to get it working. This project pushed me to deepen my understanding of React and taught me how to build more complex applications using a component-based architecture.

Techniques Learned: Lifting up state, passing props between components, BrowserRouter for routing, Github gh-pages for deployment and useRef hook.

Challenging Aspects: Lifting up state, deploying a React project that uses BrowserRouter, and installing the kit.

Difficulty: Super Hard (I relied on AI quite a bit during this project, especially when trying to deploy the application to GitHub Pages and install the required React packages. I spent a lot of time searching for guides and tutorials, but many of them either lacked clear explanations or did not match my specific setup. Getting everything configured correctly was one of the most frustrating parts when I was doing this project.)
